							Debugsim 中文说明
							=================

一、为什么写这个软件？
======================
	1. 肯定是为了好玩嘛！
	2. 为了写一个项目练习C语言。
	3. 本着造轮子的精神，网上好的模拟器很多，自己实现一个是不是觉得很赞。
	4. 学习计算机体系结构。
	5. 学习好的项目的经验，比如模块设计，文档撰写，编码规范，合作规范等...

二、模块设计:
=============

								+------------------+
								|	 Debugsim      |
								+------------------+
										 |
								+------------------+		 +-----------------+
								| Command Transter |-------->|	ISA Simulator  |
								+------------------+		 +-----------------+
										 |									   |
			   +-----+--+---------+--+-------+--+--------+--+----------+	   |
			   | Mem |  |Register |	 |IO Port|  |Assember|  |Dissemeber|	   |
			   +-----+--+---------+--+-------+--+--------+--+----------+	   |
                  |          |           |                                     |
                  +<---------+<----------+<------------------------------------+

	整个设计模块化，任何一个拿出来都能自成体系。所以里面的汇编模块和反汇编模块可以
	直接拿出来做汇编器编译8086指令集然后在 INTEL 或者 AMD 处理器在实模式下运行。

三、分工合作.
=============
	WGG 	: IO 端口模块(IO Port).
	Bobi	: 汇编器模块(Assemeber).
	Hacking	: 所有简单的模块.  :)
	
	整个设计中，当属汇编模块比较难弄, 因为涉及到汇编器的语法以及词法解析。不过我们只是使用
	INTEL汇编语法的很小很小的子集。
	
四、项目编译.
=============
	1.在Windows下设置好编译器的PATH,LIB,INCLUDE 等环境变量。在根目录有个make_vc.cmd 编译脚本，运行即可。
	2.UNix like 系统运行make_gcc.sh.
	
	建议使用Notepad++编辑器进行编码。
